Short answer

这五套数据不是同一种 benchmark：3 套直接面对真实网页，WebArena 使用可重置自托管站点，LiveClawBench 使用容器化模拟服务。规模可以并列展示，成功率与难度不能脱离环境和评测器直接比较。

02 / ENVIRONMENT

评测环境决定了结果能否横向比较。

真实网站更接近线上使用，但受页面漂移、登录态和风控影响；可控环境牺牲部分真实性，换取可复现状态与更确定的验证器。

03 / COMPOSITION

任务规模与难度标签呈现不同的组成结构。

只有 Online-Mind2Web、LexBench 与 LiveClawBench 提供可直接对齐的 easy / medium / hard 标签；WebArena 与 ClawBench 不纳入三级难度对比。

04 / DATASET LEDGER

每套数据解决的问题不同。

代表任务用于说明任务形态，不是随机抽样，也不用于估计成功率。

真实网页鲁棒性

Online-Mind2Web、LexBench、ClawBench。适合观察线上行为，但需要记录时间、地区、登录态与失败原因。

可控浏览器回归

WebArena。适合稳定比较 agent 版本，必须固定站点镜像、初始状态和 evaluator 版本。

跨工具工作流

LiveClawBench。适合测试多服务编排、污染状态、隐式目标和产物质量。

Action and boundary

后续评测应先按环境分层，再在同层内比较 agent。

真实网页组单独记录登录态、风控和页面漂移；WebArena 固定镜像与初始化状态；LiveClawBench 固定容器和 verifier 版本。不要把五套数据合成一个未经分层的总成功率。
