让云端浏览器成为可训练的 RL 环境

我们已经把 Lexmount 的隔离云端浏览器接入 NeMo-Gym 的 Resources Server，并完成了早期可用性与环境稳定性验证。接下来希望与 NVIDIA 一起把它在真正的多节点集群上跑成一条可复现的训练路径。

Reader path

公司与问题 → 原型证明 → 上游 PR → 验证设计 → 已证实与受阻 → 需要 NVIDIA 确认的条件

01 / Who we are

Lexmount 是面向智能体的云端浏览器基础设施。

我们提供可隔离、可并发、可观测的远程浏览器会话，让浏览器不必作为训练节点上的本地进程来管理。

02 / Why NeMo-Gym

NeMo-Gym 需要一个真正可交互、可隔离的浏览器环境。

Issue #644 提出了 BrowserGym 集成需求；检索型工具无法覆盖导航、点击、输入、观察和任务完成等有状态交互。

03 / First proof

第一步不是追求规模，而是证明这个环境能被训练循环正确使用。

原型把浏览器抽象成一个可插拔后端：本地 Playwright 用于 CI，Lexmount 用于远程生产/训练会话；并在 MiniWoB++ 小范围训练上看到正向学习信号。

04 / Upstream PR

PR #1865 把浏览器能力落成了一个可审阅的上游扩展。

它不是一个 demo wrapper：每条 rollout 有独立浏览器会话，工具语义与验证步骤被显式定义，并且本地与远程后端遵循同一契约。

05 / Validation plan

我们把“有效”拆成环境正确性、训练效果和资源效率三层。

早期 MiniWoB++ 只回答“能跑通”。接下来的真实站点实验必须把浏览器基础设施本身与模型学习、任务难度、集群资源分开计量。

06 / Evidence & blockers

我们已有可用证据，也清楚哪些还不能下结论。

下面把已验证实现、已测量的环境结果、正在排查的训练信号和实际阻塞的集群容量分开呈现。

07 / What we need to confirm

需要 NVIDIA 确认的不是一句“给资源”，而是一组可复现实验的运行条件。

下面的答案将决定我们交付的是自包含 batch runner，还是能与集群现场一起调试并产出完整对照证据的实验包。
