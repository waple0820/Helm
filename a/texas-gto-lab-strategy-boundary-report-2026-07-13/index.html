Texas GTO Lab · current product claim

可用于单挑训练，不能宣称多人 GTO。

这份报告回答的不是“系统会不会给建议”，而是：在已修复策略越界与部署假成功后，Texas GTO Lab 现在能向用户承诺什么、还不能承诺什么，以及下一步应把工程资源投向哪里。

报告问题
当前策略输出的可信边界与产品说法。
数据截至
2026-07-13 的本地代码、固定牌例、测试与公网复测。
结论置信度
工程边界高；单挑策略中等；多人均衡低。

短答案 / 决策

近期主线应是单挑求解质量：翻前继续使用有来源的范围表；多人翻后只提供明确标记的“范围权益近似”。在独立的多人求解、训练与可利用度验证出现前，任何“多人 GTO”商业表述都不成立。

阅读路径

先确认可对外说的策略边界，再看修复是否真的改变了行为与验证结果；随后区分这些证据能证明什么、不能证明什么，最后给出下一阶段的工程顺序。

05 / interpretation

这些证据改变了什么，没有改变什么

这是从测试结果到产品决策的推理层。它必须和原始观测分开，避免把工程事实误说成策略事实。

现在可以依赖的

  • 多人路径不会再偷偷调用单挑 exact / distilled / trained 策略。
  • 指定河牌、转牌与多加注抽象具备相应范围内的收敛证据。
  • 部署门禁能区分新构建与端口上残留的旧服务。

现在仍不能对外承诺的

  • 多人翻后是求解过的 GTO 或具有可利用度保证。
  • 单挑翻牌、转牌已经覆盖完整无限注树。
  • 36 个策略 warning 已经无害，或持续交付已经有完整外部监控。

06 / action

下一阶段：先缩小未知，再扩大能力

优先级按“消除会误导用户的风险”排序，而不是按最容易做出新功能排序。

现在
处理 36 个审计 warning。为每类 warning 固化随机种子牌例，区分 GTO 允许的近纯策略和真实偏差；每一次修改回到行动树、范围与 EV / exploitability 证据。
随后
让完整行动历史成为策略节点身份。位置、下注尺度序列、范围收缩与存活玩家数必须进入同一节点契约，不再只用 single-raised / facing-raise 粗分类。
主线
用 5090 跑单挑 solve → distill → exploitability 流水线。训练目标来自求解器 regret / average strategy，而不是人工公式；把误差与模型适用域一并产出。
独立研究
把多人作为独立研究线。先定义三人有限尺度树、范围抽象和可利用度评估；指标达标前持续使用“多人参考”标签。
交付门禁
补 CI 与外部可观测性。PR 自动跑核心审计与快速求解器自测；部署记录 commit、资源哈希、进程 owner、启动时间，并增加持续可用性检查。
最后
性能与商业包装后置。先拆分 2.98 MB 主包（gzip 约 791 KB），再讨论付费、账户、训练档案和扩容。

07 / boundary

发布与商业化的硬边界

这是一条产品约束，不是谨慎措辞：不要让交互界面、教练文案或付费页超出实际证据。

Revisit this decision if

当多人有限树已经拥有独立 CFR / Nash 求解、训练集、固定回归与可利用度阈值验证后，才重新评估多人标签；当 36 个 warning 被逐类解释或修复、CI 与外部监控闭环后，才重新评估商业化和更大规模部署。不要为了让 warning 数字下降而机械加入混合频率。
